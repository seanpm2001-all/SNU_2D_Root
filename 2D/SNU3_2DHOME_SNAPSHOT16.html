What is the embedded directory? What is the high directory? Why does it matter?

Originally, all test files for SNU were in a single directory. It has gotten hard to manage, so more directories have been created. The embedded directory is the original directory, which is stored in the same directory as the current document.

The high directory is the directory within the directory. It is the new spot for the test files, as it is easier to access this way. When servers get large, they have to migrate a little bit.

Both directories do the same thing, just in a different location. It doesn't really matter which one you choose.
